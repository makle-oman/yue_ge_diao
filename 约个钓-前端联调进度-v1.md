## 4. 关键约定

### 4.1 响应信封

后端 `TransformInterceptor` 统一返回：

```json
{ "code": 200, "msg": "ok", "data": { ... }, "traceId": "xxxx" }
```

- **约定：`code === 200` 表示业务成功，与 HTTP 200 同义；其他任何值（包括 0）一律视为失败。**
- 前端 `request.ts` 在 `code === 200` 时把 `data` 透出，调用方只看到干净的数据。
- `code !== 200` → 抛 `BizError(code, msg)`；调用方可 `instanceof BizError` 分支处理，或交给默认 Toast。
- 异常路径由 `AllExceptionFilter` 写出（如 `code:500`、`code:401`、自定义业务码），都会被前端识别为失败。

### 4.2 Trace ID

- 前端每个请求生成短 ID `c_xxxxxx` 放 `X-Trace-Id`。
- 后端 `TraceIdMiddleware` 优先用 header 里的值，否则自己生成 uuid。
- 后端日志、异常响应都会回带 `traceId`，便于跨端定位。

### 4.3 鉴权

- 全局 `JwtAuthGuard` 默认拦截，公开接口需在 controller 方法上加 `@Public()`。
- 当前公开接口：`POST /common/config`、`POST /auth/dev-login`。
- 受保护接口：`POST /users/me`，以及后续所有业务接口（默认）。

### 4.4 baseUrl

- 开发：`http://localhost:3000/api`
- 生产：`https://api.yuegediao.example.com/api`（占位，发布前改）
- 切换由 `process.env.NODE_ENV` 自动判别。
- 后端已 `app.enableCors({ origin: true, credentials: true })`，H5 本地直连无需 Vite 代理。

## 5. 联调验证步骤

### 5.1 启动后端

```bash
cd yue-ge-diao-infra
docker compose up -d           # 起 MySQL
cd ../yue-ge-diao-server
pnpm install                   # 首次
pnpm prisma generate
pnpm prisma db push            # 首次或 schema 改动后
pnpm start:dev                 # 监听 3000
```

预期：
- 控制台打印 `🚀 yue-ge-diao-server listening on http://localhost:3000/api`
- `curl -X POST http://localhost:3000/api/common/config` 返回 `code:200` 信封

### 5.2 启动前端 H5

```bash
cd yue-ge-diao
pnpm install
pnpm dev:h5                    # Vite 起在 5173 或自动分配
```

### 5.3 走主链路

1. 浏览器打开 H5 入口，进入「我的」Tab → 因未登录会被 `request.ts` 拦到 401 并跳到 `/pages/login/index`（首次访问也可以从其他入口手动进 login 页）。
2. 勾选协议 → 点「[开发] 随机账号登录」。
3. 期望：
   - Toast「登录成功」
   - 自动跳到「我的」Tab
   - 顶部昵称显示后端 upsert 出来的 `钓友xxxx`（4 位 openid 尾号）
   - 浏览器 DevTools → Application → Local Storage 能看到 `yg_token` / `yg_user`
4. 在「我的」页点底部「退出登录」→ 弹确认 → Toast「已退出」→ 跳回 login 页 → 本地 `yg_token` 已清空。

### 5.4 401 自动跳登录

- 在 DevTools 把 `yg_token` 改成乱码后刷新「我的」Tab。
- 期望：Toast「登录已过期…」+ 自动 redirect 到 `/pages/login/index`，且本地 `yg_token` / `yg_user` 已被清。

## 7. 业务模块接入模板

新加业务模块时，照搬：

1. 后端：`modules/<x>/<x>.controller.ts` 用 `@Post('<action>')`，service 返回原始对象，无需自己 wrap 信封。
2. 前端：在 `src/api/<x>.ts` 写 `export function fooBar(params): Promise<XxxResp> { return http.post<XxxResp>('/x/<action>', params); }`。
3. 页面：`import { fooBar } from '@/api/x'` → `await fooBar({...})`；错误由 request 层兜底，业务想覆盖就 try/catch。
