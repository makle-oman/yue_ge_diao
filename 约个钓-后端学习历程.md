## 四、疑问追踪

> 学的时候随时记。**问完了把答案写在下面**，下次自己能复盘。

### Q1: （示例）`let mut` 为什么要加 `mut`？

**问的时候在想**：JavaScript `let` 直接就能改，Rust 为啥还要再标一遍？

**答案**（自己消化后写）：

> Rust 默认所有变量**不可变（immutable）**——这是为了让你"看代码就知道这个变量会不会变"，省去追踪心智。`mut` 是显式标记"我打算改它"。这种"默认安全、要破坏需要明说"的设计在 Rust 里到处都是。

**关联概念**：immutability、shadowing（`let x = x + 1;` 重新绑定）

## 七、命令速查

### Cargo

```bash
cargo new <name>             # 新建项目
cargo new <name> --lib       # 新建库（无 main.rs）
cargo run                    # 编译并运行
cargo build                  # 编译（不跑）
cargo build --release        # 优化编译（慢，但跑得快，用于发布）
cargo check                  # 只做类型检查，不生成二进制，快
cargo test                   # 跑所有测试
cargo test <name>            # 只跑名字含 <name> 的测试
cargo clippy                 # 静态检查（Rust 版 ESLint）
cargo fmt                    # 格式化代码
cargo update                 # 更新依赖
cargo add <crate>            # 加依赖（替代手动改 Cargo.toml）
cargo doc --open             # 生成 + 打开本项目所有依赖的文档
cargo tree                   # 看依赖树
```

### Docker（Redis）

```bash
# 启动 Redis（首次）
docker run -d --name redis -p 6379:6379 redis:7-alpine

# 之后启停
docker start redis
docker stop redis

# 查看运行中的容器
docker ps

# 看日志
docker logs -f redis

# 进 redis-cli
docker exec -it redis redis-cli
> PING                       # 应回 PONG
> KEYS *
> GET some-key
```

### MySQL

```bash
# 命令行登录
mysql -u yue -p yue_ge_diao

# 常用 SQL
SHOW DATABASES;
USE yue_ge_diao;
SHOW TABLES;
DESC users;                  # 看表结构
SELECT * FROM users LIMIT 5;
```

### sea-orm-cli（W2 后用到）

```bash

  -o src/entity
```

## 九、给未来的自己的话

> 当你在 ch4 所有权那一章想砸键盘的时候，回来看这里：

- 不懂是**正常**的。这章 90% 的 Rust 学习者都要读 2-3 遍。
- 不要跳。跳过所有权，后面所有报错你都看不懂。
- 实在卡住，就用 `clone()` 暴力让代码先跑通——**先跑起来再优化**比"想到完美方案再写"重要 10 倍。
- 编译器报错信息**是金矿**，认真读完，它会告诉你怎么改。
- 想放弃的时候记得：你不是在学一门"工具"，你在重塑你写代码时大脑里的"内存模型"。这个收益是终身的。
