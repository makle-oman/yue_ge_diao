### 1.2 一些被舍弃的选项及理由

| 候选 | 为什么不选 |
|---|---|
| Express 裸用 | NestJS 已封装它；DI/Module/装饰器更适合复杂业务 |
| Koa / Fastify 裸用 | 同上；个人 MVP 不需要榨性能 |
| Sequelize | 老牌但类型推断弱，TS 体验不如 Prisma/TypeORM |
| MikroORM | UoW 模式优雅但学习曲线略陡 |
| MongoDB | 关系建模本来就清晰，没必要 |
| 腾讯云 COS | 用户阿里云生态，统一到 OSS 减少多账号/多 SDK 负担 |
| Bun / Deno | 生态仍在追赶，NestJS 在 Node 上最稳 |

### 1.3 ORM 选型：Prisma vs TypeORM（详细）

| 维度 | **Prisma**（推荐） | TypeORM |
|---|---|---|
| 风格 | Schema-first（独立 DSL） | Decorator-first（实体类即 schema） |
| 类型推断 | 由 Prisma Client 自动生成，极佳 | 装饰器+泛型，偶尔类型丢失 |
| 迁移工具 | `prisma migrate dev/deploy`，命令一致 | 自带迁移但生成 SQL 不够干净，需要手改 |
| 关系查询 | `include` / `select` 显式，写法清晰 | `relations: []` + QueryBuilder，灵活但啰嗦 |
| 原生 SQL | `prisma.$queryRaw` 安全模板字符串 | `manager.query()` |
| 与 NestJS 集成 | 用 `PrismaService extends PrismaClient` 自己封一层即可（社区有 nestjs-prisma 包） | `@nestjs/typeorm` 官方包，开箱即用 |
| **缺点** | 地理类复杂查询（geohash IN 多个值）走 `$queryRaw`；不支持复合主键的部分场景 | 类型推断和迁移体验比 Prisma 弱 |

**结论**：v1 推荐 **Prisma**，理由：
1. 你是前端转后端，Schema-first 一份文件能 grep 出所有表结构 + 关系，心智负担最低
2. Prisma Client 的类型推断质量直接决定开发速度
3. Migration 流程简单（`prisma migrate dev` 自动生成 + 应用 + 重新生成 client）
4. v1 唯一需要原生 SQL 的就是"geohash IN 9 个前缀的附近查询"，可控

> 💡 **如果你坚持 TypeORM**：换成它的成本就是 §五 的代码示例形态变化，数据库/接口设计本身不变。我可以在 §八 等你确认后单独再出一版 TypeORM 实现示例。

- Redis（可选）：`maxmemory=128mb`，allkeys-lru 策略

## 二、API 接口规范（**全 POST、RPC 风格**）

### 2.1 统一约定

- **方法**：一律 `POST`
- **路径**：`/api/{module}/{action}`，例如 `/api/spots/list`
- **入参**：JSON Body（`Content-Type: application/json`），统一 **camelCase**
- **鉴权**：Header `Authorization: Bearer <jwt>`；少数白名单接口免鉴权
- **统一响应**：

  ```json
  {
    "code": 0,
    "msg": "ok",
    "data": { },
    "traceId": "abc123"
  }
  ```
  业务错误用非 0 `code`；HTTP 状态码统一 200（除鉴权 401）。

- **分页约定**：cursor-based，回 `{ list, nextCursor, hasMore }`
- **字段命名**：前后端统一 **camelCase**（数据库 snake_case，由 Prisma 自动映射）

### 2.2 NestJS Controller 示例

```ts
// src/modules/spots/spots.controller.ts
import { Body, Controller, Post, UseGuards } from '@nestjs/common';
import { JwtAuthGuard } from '../auth/jwt.guard';
import { CurrentUser } from '../auth/current-user.decorator';
import { SpotsService } from './spots.service';
import { NearbyReqDto, NearbyRespDto } from './dto/nearby.dto';

@Controller('api/spots')
@UseGuards(JwtAuthGuard)
export class SpotsController {
  constructor(private readonly spots: SpotsService) {}

  @Post('nearby')
  nearby(
    @CurrentUser() userId: number,
    @Body() dto: NearbyReqDto,
  ): Promise<NearbyRespDto> {
    return this.spots.nearby(userId, dto);
  }

  @Post('list')
  list(@Body() dto: ListReqDto) { /* ... */ }

  @Post('create')
  create(@CurrentUser() userId: number, @Body() dto: CreateReqDto) { /* ... */ }

  // get / update / delete / want ...
}
```

> 💡 **NestJS 三大基石**：
> - **Module**：用 `@Module({ controllers, providers, imports })` 声明依赖图，让 IoC 容器知道怎么装配
> - **Controller**：路由 + HTTP 入口，通过构造器注入 Service
> - **Provider/Service**：业务逻辑实现，靠 `@Injectable()` 让容器知道"我可以被 inject"

### 3.2 地理查询方案：geohash + 边界框

**核心思想**：把 `(lat, lng)` 二维问题降维成"字符串前缀匹配"的一维问题。

- 把地球递归切成网格。每往字符串里加一个字符，网格精度提高一档。
- 精度 6 ≈ `1.2km × 0.6km`；精度 7 ≈ `153m × 153m`；精度 8 ≈ `38m × 19m`。
- 两个相邻点 geohash 前缀大概率相同，但**网格边界处**会失效 —— 所以查附近要查"自己 + 周围 8 格"共 9 个前缀。
- 应用层把 9 个前缀 OR 起来查 MySQL，**再做精确距离过滤**。

> 💡 Prisma 不直接支持 geohash 函数，但 9 个前缀的 `IN (...)` 是普通字符串查询，可以直接用 `prisma.spot.findMany({ where: { geohashPrefix: { in: prefixes } } })` —— 我们把 geohash 字段存定长 `CHAR(8)`，查询时按精度截前 N 位。

### 5.2 统一响应包装

```ts
// src/common/interceptors/transform.interceptor.ts
import { CallHandler, ExecutionContext, Injectable, NestInterceptor } from '@nestjs/common';
import { map, Observable } from 'rxjs';

export interface ApiResp<T> {
  code: number;
  msg: string;
  data: T | null;
  traceId: string;
}

@Injectable()
export class TransformInterceptor<T> implements NestInterceptor<T, ApiResp<T>> {
  intercept(ctx: ExecutionContext, next: CallHandler<T>): Observable<ApiResp<T>> {
    const req = ctx.switchToHttp().getRequest();
    return next.handle().pipe(
      map((data) => ({
        code: 0,
        msg: 'ok',
        data: data ?? null,
        traceId: req.traceId,
      })),
    );
  }
}
```

### 5.3 业务异常 & 全局异常过滤器

```ts
// src/common/exceptions/business.exception.ts
export class BusinessException extends Error {
  constructor(public readonly code: number, message: string) {
    super(message);
  }
}

// src/common/filters/all-exception.filter.ts
import { ArgumentsHost, Catch, ExceptionFilter, HttpException, Logger } from '@nestjs/common';
import { Prisma } from '@prisma/client';
import { BusinessException } from '../exceptions/business.exception';

@Catch()
export class AllExceptionsFilter implements ExceptionFilter {
  private readonly logger = new Logger(AllExceptionsFilter.name);

  catch(exception: unknown, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const res = ctx.getResponse();
    const req = ctx.getRequest();
    const traceId = req.traceId;

    let httpStatus = 200;
    let code = 500;
    let msg = '服务器异常';

    if (exception instanceof BusinessException) {
      code = exception.code;
      msg = exception.message;
    } else if (exception instanceof HttpException) {
      httpStatus = exception.getStatus();
      code = httpStatus === 401 ? 401 : httpStatus;
      msg = exception.message;
    } else if (exception instanceof Prisma.PrismaClientKnownRequestError) {
      this.logger.error(`prisma ${exception.code}: ${exception.message}`);
      msg = '数据库异常';
    } else {
      this.logger.error(exception);
    }

    res.status(httpStatus).json({ code, msg, data: null, traceId });
  }
}
```

> 💡 业务里这样抛：`throw new BusinessException(40001, '钓点已存在')`，filter 自动转成 `{code:40001, msg:'钓点已存在', ...}`。HTTP 状态码统一 200（除了真正的鉴权 401），方便小程序端处理。

import { AuthGuard } from '@nestjs/passport';
import { Reflector } from '@nestjs/core';
import { IS_PUBLIC_KEY } from '../decorators/public.decorator';

### 5.6 附近钓点 `POST /api/spots/nearby`（geohash 完整方案）

```ts
// src/infra/geohash/geohash.util.ts
const BASE32 = '0123456789bcdefghjkmnpqrstuvwxyz';

export function encode(lat: number, lng: number, precision = 8): string {
  let latRange: [number, number] = [-90, 90];
  let lngRange: [number, number] = [-180, 180];
  let bits = 0;
  let bitCount = 0;
  let even = true;
  let out = '';

  while (out.length < precision) {
    const mid = even ? (lngRange[0] + lngRange[1]) / 2 : (latRange[0] + latRange[1]) / 2;
    const target = even ? lng : lat;
    if (target >= mid) {
      bits = (bits << 1) | 1;
      if (even) lngRange[0] = mid; else latRange[0] = mid;
    } else {
      bits <<= 1;
      if (even) lngRange[1] = mid; else latRange[1] = mid;
    }
    even = !even;
    bitCount++;
    if (bitCount === 5) {
      out += BASE32[bits];
      bits = 0;
      bitCount = 0;
    }
  }
  return out;
}

export function neighbors(lat: number, lng: number, precision: number): string[] {
  const cell: Record<number, number> = { 5: 0.044, 6: 0.0055, 7: 0.00068, 8: 0.000086 };
  const d = cell[precision] ?? 0.001;
  const set = new Set<string>();
  for (const dLat of [-d, 0, d])
    for (const dLng of [-d, 0, d])
      set.add(encode(lat + dLat, lng + dLng, precision));
  return [...set];
}

export function distanceM(lat1: number, lng1: number, lat2: number, lng2: number): number {
  const R = 6_371_000;
  const toRad = (x: number) => (x * Math.PI) / 180;
  const dLat = toRad(lat2 - lat1);
  const dLng = toRad(lng2 - lng1);
  const a = Math.sin(dLat / 2) ** 2 +
            Math.cos(toRad(lat1)) * Math.cos(toRad(lat2)) * Math.sin(dLng / 2) ** 2;
  return 2 * R * Math.asin(Math.sqrt(a));
}
```

```ts
// src/modules/spots/spots.service.ts
@Injectable()
export class SpotsService {
  constructor(private readonly prisma: PrismaService) {}

  async nearby(userId: number, dto: NearbyReqDto) {
    const radius = dto.radius ?? 5000;
    const precision = radius <= 500 ? 7 : 6;
    const prefixes = neighbors(dto.lat, dto.lng, precision);

    // 因为 geohash 字段存的是 precision=8，前缀匹配走 LIKE 的话索引不友好。
    // 优化：在 prisma where 里用 IN 比较前 N 位 —— 通过 $queryRaw 一次性拿。
    const rows = await this.prisma.$queryRaw<Spot[]>`
      SELECT id, name, type, lat, lng, geohash, photos, avg_rating AS avgRating, want_count AS wantCount
      FROM spots
      WHERE status = 'approved'
        AND LEFT(geohash, ${precision}) IN (${Prisma.join(prefixes)})
    `;

    const list = rows
      .map((s) => ({
        ...s,
        distance: distanceM(dto.lat, dto.lng, Number(s.lat), Number(s.lng)),
      }))
      .filter((s) => s.distance <= radius)
      .sort((a, b) => a.distance - b.distance)
      .slice(0, dto.limit ?? 50);

    return { list };
  }
}
```

> 💡 这是 v1 唯一使用 `$queryRaw` 的地方。如果后续切到 PostgreSQL，可以换成 PostGIS 的 `ST_DWithin`，但 MySQL 8 也有 spatial 函数（`ST_Distance_Sphere`），先 geohash + 应用层过滤就够。
