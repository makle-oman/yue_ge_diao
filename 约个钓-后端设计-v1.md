### 1.2 一些被舍弃的选项及理由

| 候选 | 为什么不选 |
|---|---|
| Actix-web | 性能略快但 API 接口不稳；社区分裂；Tokio 生态隔离 |
| Rocket | 更新慢；async 支持迟到 |
| SQLx 裸用 | 编译期校验 SQL 很爽但对新人陡峭；SeaORM 内部就用 SQLx |
| Diesel | 同步为主，async 是补丁；学习曲线对新人更陡 |
| MongoDB | 关系建模本来就清晰，没必要 |
| Loco（Rails-like 全家桶） | 一站式但隐藏过多，学习反而不透 |

> 💡 **学习提示**：Rust 没有 "Spring Boot / NestJS" 那种"开箱即用大全套"。社区文化是**自己组合工具箱**——Axum 只做 Web 层，数据库要单独选，连日志都要自己装。这种"积木式开发"前两周让人崩溃，但通透之后你会知道"每个齿轮在哪里"。

### 1.3 部署架构（MVP）

```
[微信小程序 / H5]

       ▼
[Nginx 80/443] (反向代理 + TLS 终止)
       │
       ▼
[Rust 二进制 :3000] ──→ [Redis :6379] (缓存/限流/会话)
```

## 二、API 接口规范（**全 POST、RPC 风格**）

### 2.1 统一约定

约定与原方案一致：
- **方法**：一律 `POST`
- **路径**：`/api/{module}/{action}`，例如 `/api/spots/list`
- **入参**：JSON Body（`Content-Type: application/json`）
- **鉴权**：Header `Authorization: Bearer <jwt>`
- **统一响应**：

  ```json
  {
    "code": 0,
    "msg": "ok",
    "data": { },
    "trace_id": "abc123"
  }
  ```
  业务错误用非 0 `code`；HTTP 状态码统一 200（除鉴权 401）。

- **分页约定**：cursor-based，回 `{ list, next_cursor, has_more }`
- **字段命名**：JSON 用 `snake_case`（Rust 默认风格；前端 axios interceptor 转 camelCase 即可）

### 2.2 Axum 路由示例

> 💡 这段代码先扫一眼，建立画面感即可，不需要现在看懂每一行。

```rust
// src/modules/spots/router.rs
use axum::{routing::post, Router};
use crate::AppState;

pub fn router() -> Router<AppState> {
    Router::new()
        .route("/nearby",  post(handlers::nearby))
        .route("/list",    post(handlers::list))
        .route("/get",     post(handlers::get))
        .route("/create",  post(handlers::create))
        .route("/update",  post(handlers::update))
        .route("/delete",  post(handlers::delete))
        .route("/want",    post(handlers::want))
}

// src/main.rs 里挂载
let app = Router::new()
    .nest("/api/auth",   auth::router())
    .nest("/api/users",  users::router())
    .nest("/api/spots",  spots::router())
    .nest("/api/catches", catches::router())
    // ...
    .layer(middleware::auth_layer())
    .with_state(state);
```

> 💡 **学习提示**：`Router::new().route(...)` 是**链式构造器模式**——每个 `.route()` 返回新的 `Router`，所有权被 move 进下一次调用。这种"消耗 self 返回 Self"的模式（Builder Pattern）在 Rust 里无处不在，因为它和所有权系统天然契合。

### 3.2 地理查询方案：geohash + 边界框

**核心思想**：把 `(lat, lng)` 二维问题降维成"字符串前缀匹配"的一维问题。

- 把整个地球递归切成网格。每往字符串里加一个字符，网格精度提高一档。
- 精度 6 的 geohash 网格约 `1.2km × 0.6km`；精度 7 约 `153m × 153m`。
- 两个相邻的点 geohash 前缀大概率相同，但**网格边界处**会失效——所以查附近时需要查"自己 + 周围 8 格"共 9 个前缀。
- 应用层把 9 个前缀 OR 起来查 MySQL，**再做精确距离过滤**。

### 3.3 索引策略要点

| 查询场景 | 走的索引 |
|---|---|
| 附近钓点 | `idx_geohash_status`（应用层算 9 个 geohash 前缀，IN 查询） |
| 我的鱼获 / 钓点鱼获 | `idx_user_time` / `idx_spot_time` |
| Feed 时间线 | `idx_feed`（复合 `(review_status, created_at)`） |
| 未读消息 | `idx_user_unread`（`read_at IS NULL` 自动走前缀） |
| 关注 / 点赞 / 想去 | 复合主键，天然唯一约束 |

> 💡 **学习提示**：MySQL 索引的核心规律是"**最左前缀匹配**"。`(geohash, status)` 索引能加速 `WHERE geohash=? AND status=?`、`WHERE geohash=?`，但**加速不了** `WHERE status=?`。

### 5.2 统一响应包装

```rust
// src/response.rs
use serde::Serialize;

#[derive(Serialize)]
pub struct ApiResp<T: Serialize> {
    pub code: i32,
    pub msg: String,
    pub data: Option<T>,
    pub trace_id: String,
}

impl<T: Serialize> ApiResp<T> {
    pub fn ok(data: T) -> Self {
        Self { code: 0, msg: "ok".into(), data: Some(data), trace_id: tracing_id() }
    }
    pub fn fail(code: i32, msg: impl Into<String>) -> ApiResp<()> {
        ApiResp { code, msg: msg.into(), data: None, trace_id: tracing_id() }
    }
}

fn tracing_id() -> String {
    // 从 tracing::Span 里取 trace_id；MVP 阶段简单实现
    uuid::Uuid::new_v4().to_string()
}
```

💡 **学习点**：
- `<T: Serialize>` 是**泛型 + trait bound**——T 必须实现 `serde::Serialize` 才能 JSON 序列化
- `impl Into<String>` 让调用方可以传 `&str`、`String`、`Cow<str>` 等任何能转 String 的类型，比单纯写 `String` 灵活

### 5.3 错误处理

```rust
// src/error.rs
use axum::{http::StatusCode, response::{IntoResponse, Response}, Json};
use crate::response::ApiResp;

#[derive(Debug, thiserror::Error)]
pub enum AppError {
    #[error("unauthorized")]
    Unauthorized,
    #[error("forbidden")]
    Forbidden,
    #[error("not found")]
    NotFound,
    #[error("bad request: {0}")]
    BadRequest(String),
    #[error("business: {code} {msg}")]
    Business { code: i32, msg: String },
    #[error("database: {0}")]
    Db(#[from] sea_orm::DbErr),
    #[error("redis: {0}")]
    Redis(#[from] redis::RedisError),
    #[error("internal: {0}")]
    Internal(#[from] anyhow::Error),
}

impl IntoResponse for AppError {
    fn into_response(self) -> Response {
        let (status, code, msg) = match &self {
            AppError::Unauthorized => (StatusCode::UNAUTHORIZED, 401, "unauthorized".into()),
            AppError::Forbidden    => (StatusCode::OK, 403, "forbidden".into()),
            AppError::NotFound     => (StatusCode::OK, 404, "not found".into()),
            AppError::BadRequest(m) => (StatusCode::OK, 400, m.clone()),
            AppError::Business { code, msg } => (StatusCode::OK, *code, msg.clone()),
            AppError::Db(e) => {
                tracing::error!("db error: {e:?}");
                (StatusCode::OK, 500, "服务器异常".into())
            }
            AppError::Redis(e) => {
                tracing::error!("redis error: {e:?}");
                (StatusCode::OK, 500, "服务器异常".into())
            }
            AppError::Internal(e) => {
                tracing::error!("internal: {e:?}");
                (StatusCode::OK, 500, "服务器异常".into())
            }
        };
        (status, Json(ApiResp::<()>::fail(code, msg))).into_response()
    }
}

pub type AppResult<T> = Result<T, AppError>;
```

💡 **学习点**：
- `thiserror::Error` 是个派生宏，自动给 enum 生成 `Display`/`Error` 实现，省去 `match` 写 `Display` 的样板
- `#[from]` 让 `?` 运算符可以自动把 `sea_orm::DbErr` 转成 `AppError::Db`——这是 **From trait** 自动派生
- `IntoResponse` 是 Axum 的核心 trait——任何实现它的类型都能作为 handler 的返回值。**自定义错误类型只要实现这一个 trait，就能被 `?` 自然抛出并自动包成 HTTP 响应**

### 5.5 附近钓点 `POST /api/spots/nearby`（geohash 方案完整代码）

```rust
// src/util/geohash.rs
const BASE32: &[u8] = b"0123456789bcdefghjkmnpqrstuvwxyz";

pub fn encode(lat: f64, lng: f64, precision: usize) -> String {
    let mut lat_range = (-90.0_f64, 90.0_f64);
    let mut lng_range = (-180.0_f64, 180.0_f64);
    let mut bits = 0u8;
    let mut bit_count = 0;
    let mut even = true; // true 表示当前位编码经度
    let mut out = String::with_capacity(precision);

    while out.len() < precision {
        let mid = if even {
            (lng_range.0 + lng_range.1) / 2.0
        } else {
            (lat_range.0 + lat_range.1) / 2.0
        };
        let target = if even { lng } else { lat };

        if target >= mid {
            bits = (bits << 1) | 1;
            if even { lng_range.0 = mid; } else { lat_range.0 = mid; }
        } else {
            bits <<= 1;
            if even { lng_range.1 = mid; } else { lat_range.1 = mid; }
        }
        even = !even;
        bit_count += 1;

        if bit_count == 5 {
            out.push(BASE32[bits as usize] as char);
            bits = 0;
            bit_count = 0;
        }
    }
    out
}

/// 返回该点 geohash + 周围 8 邻居（共 9 个前缀）
pub fn neighbors(lat: f64, lng: f64, precision: usize) -> Vec<String> {
    // 简化实现：在 (lat, lng) 四周加减一个网格高/宽，编码后去重
    let cell_size = match precision {
        5 => 0.044, 6 => 0.0055, 7 => 0.00068, 8 => 0.000086, _ => 0.001,
    };
    let mut set = std::collections::HashSet::new();
    for d_lat in [-cell_size, 0.0, cell_size] {
        for d_lng in [-cell_size, 0.0, cell_size] {
            set.insert(encode(lat + d_lat, lng + d_lng, precision));
        }
    }
    set.into_iter().collect()
}

/// Haversine 距离公式，返回米
pub fn distance_m(lat1: f64, lng1: f64, lat2: f64, lng2: f64) -> f64 {
    let r = 6_371_000.0;
    let (lat1, lat2) = (lat1.to_radians(), lat2.to_radians());
    let d_lat = lat2 - lat1;
    let d_lng = (lng2 - lng1).to_radians();
    let a = (d_lat / 2.0).sin().powi(2) + lat1.cos() * lat2.cos() * (d_lng / 2.0).sin().powi(2);
    2.0 * r * a.sqrt().asin()
}

// src/modules/spots/service.rs
pub async fn nearby(state: &AppState, req: NearbyReq) -> AppResult<NearbyResp> {
    let radius = req.radius.unwrap_or(5000.0);
    let precision = if radius <= 500.0 { 7 } else { 6 };
    let prefixes = util::geohash::neighbors(req.lat, req.lng, precision);

    // 一次性查 9 个前缀对应的钓点
    use sea_orm::sea_query::Expr;
    let candidates = spots::Entity::find()
        .filter(spots::Column::Status.eq("approved"))
        .filter(Expr::cust("LEFT(geohash, ?) IN (?)")
                .bind(precision as i32)
                .bind(prefixes.clone()))
        .all(&state.db)
        .await?;

    // 应用层精确过滤 + 排序
    let mut results: Vec<NearbyItem> = candidates
        .into_iter()
        .filter_map(|s| {
            let d = util::geohash::distance_m(req.lat, req.lng, s.lat.to_f64(), s.lng.to_f64());
            if d <= radius {
                Some(NearbyItem::from((s, d)))
            } else { None }
        })
        .collect();
    results.sort_by(|a, b| a.distance.partial_cmp(&b.distance).unwrap());
    results.truncate(req.limit.unwrap_or(50));

    Ok(NearbyResp { list: results })
}
```

💡 **学习点**：
- `Vec<String>` 的 `.into_iter()` 拿走所有权，`.filter_map()` 同时过滤 + 映射，`.collect()` 收集成新集合——这是 Rust 函数式 API 的核心
- `f64::to_radians()` / `.sin()` / `.powi(2)` 是标准库方法链，类型推断+方法调用替代了"调用一堆全局函数"

```rust
pub async fn fishing_index(state: &AppState, req: IndexReq) -> AppResult<IndexResp> {
    let geohash = util::geohash::encode(req.lat, req.lng, 5);
    let date_key = req.date.unwrap_or_else(|| chrono::Utc::now().format("%Y%m%d%H").to_string());
    let cache_key = format!("weather:idx:{geohash}:{date_key}");

    // 1. 先查 Redis
    let mut conn = state.redis.clone();
    if let Ok(Some(cached)) = redis::cmd("GET").arg(&cache_key)
        .query_async::<_, Option<String>>(&mut conn).await
    {
        if let Ok(resp) = serde_json::from_str::<IndexResp>(&cached) {
            return Ok(resp);
        }
    }

    // 2. miss → 调和风天气
    let forecast = infra::weather::fetch_24h(&state.config, req.lat, req.lng).await?;

    // 3. 算分
    let score = score(&forecast);
    let resp = IndexResp {
        score,
        factors: factors(&forecast),
        advice: advice_text(score, &forecast),
    };

    // 4. 写回 Redis，TTL 1 小时
    let _: () = redis::cmd("SETEX").arg(&cache_key).arg(3600)
        .arg(serde_json::to_string(&resp)?)
        .query_async(&mut conn).await?;

    Ok(resp)
}

fn score(f: &Forecast) -> u32 {
    let s = 0.30 * pressure_score(f)
          + 0.20 * temp_score(f)
          + 0.20 * wind_score(f)
          + 0.15 * precip_score(f)
          + 0.15 * moon_score(f);
    (s * 100.0).round().clamp(0.0, 100.0) as u32
}
```

```rust
use axum::{extract::Request, middleware::Next, response::Response};
use crate::{state::AppState, error::AppError, util::jwt};

pub async fn require_auth(
    State(state): State<AppState>,
    mut req: Request,
    next: Next,
) -> Result<Response, AppError> {
    // 白名单
    let path = req.uri().path();
    if matches!(path,
        "/api/auth/wx-login" | "/api/auth/refresh" | "/api/common/config"
    ) {
        return Ok(next.run(req).await);
    }

    // 取 token
    let token = req.headers()
        .get("authorization")
        .and_then(|v| v.to_str().ok())
        .and_then(|s| s.strip_prefix("Bearer "))
        .ok_or(AppError::Unauthorized)?;

        .map_err(|_| AppError::Unauthorized)?;

    // 注入到 request extension，handler 用 Extension<CurrentUser> 提取
    req.extensions_mut().insert(CurrentUser { id: claims.sub });
    Ok(next.run(req).await)
}
```

- **Rust Cookbook**：常见任务的小代码

### 9.3 调试套路

| 错误信息 | 一般是 |
|---|---|
| `cannot move out of borrowed content` | 你把别人借给你的值搬走了；要 `.clone()` 或 `.to_owned()` |
| `lifetime may not live long enough` | 引用活得不够久；加 `'static` 或改用 owned 类型 |
| `the trait bound ... is not satisfied` | 类型没实现需要的 trait；查文档加 derive 或手动 impl |
| `temporary value dropped while borrowed` | 链式调用太长，中间值被丢；先 `let` 绑一下 |
| `future cannot be sent between threads safely` | 非 Send 类型跨 await 持有；查是哪个值，要么 drop 它要么换 |

### 9.4 心态

- **借用检查器不是敌人**，是个特别严的代码 reviewer。它拒绝你的代码 90% 情况下是真的有问题
- **第一周很慢是正常的**，第三周开始你会觉得编译器在帮你
- **不要追求一遍写对**，先 `clone()` 暴力让代码跑通，再回过头消除 clone
- **看 `cargo clippy` 的建议**——它是 Rust 的 ESLint，会指出更地道的写法

## 十、文档以外，开发时的工作流约定

1. **每写一个接口**：先在 `dto.rs` 写请求/响应 struct，再在 `handlers.rs` 写空壳，再写 `service.rs`，最后写 `tests.rs`
2. **每改一次表**：写 migration 文件 → `sea-orm-cli migrate up` → `sea-orm-cli generate entity` 重新生成 entity
3. **Git commit 信息**：`feat(spots): nearby query with geohash` / `fix(auth): jwt expire check` 这种 Conventional Commits 风格
4. **每个接口写完用 Bruno 跑一次**，把请求文件 commit 进 `docs/api/` 文件夹，下次回归就有用例
5. **遇到不会的随时问我**——但**问的时候给完整报错和你看不懂的那段代码**，别截个图就问"为啥不能编译"
