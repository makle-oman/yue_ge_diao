## 二、v1 设计稿缺失(需补图，按紧急度)

### 🔴 阻塞开发 — 必须补
1. **评论列表 / 回复页**(鱼获详情只放了 1 行摘要)
2. **图片大图查看器**(左右切、双指缩放)
3. **搜索结果页 + 搜索历史**(地图首页有搜索框，落地缺失)
4. **筛选弹层**(地图 chips 的"更多筛选"展开态)
5. **组队报名后状态页**(待审核/通过/拒绝/退出，我方视角)

### 🟡 流程闭环 — 建议补
6. **聊天对话页**(消息中心列表落地)
7. **钓点审核中/被拒页**(05 提示"等待审核"但缺反馈)
8. **绑定手机号 / 实名认证**(发布前合规)
9. **举报 / 拉黑**对话框
10. **禁渔期 / 禁渔区警示弹层**(法律风险)

### 🟢 异常状态 — 上线前补
11. **空态**：无钓点 / 无组队 / 无消息 / 无关注
12. **错误态**：定位失败 / 网络异常 / 审核被拒
13. **加载 / 骨架屏**

### ⚪ 长尾 — 视进度
14. 我的鱼获列表 / 我的钓点列表 / 关注 / 粉丝列表
15. 鱼类图鉴详情(11 我的有入口)
16. 钓点历史鱼获完整列表(06 只放了 3 张缩略图)
17. 数据周报/月报、爽约记录公示

## 三、v2 需求规划(按 ROI 排序，**商业化暂缓**)

### 🔥 必做 — 让一版"能闭环"

| # | 需求 | 备注 |
|---|---|---|
| 1 | 评论与回复系统 | 完整列表 + 二级回复 + @ |
| 2 | 完整通知体系 | 微信订阅消息(报名通过/被点赞/被评论/组队提醒) |
| 3 | 空态/错误态/骨架屏 | 全局体验基线 |
| 4 | 禁渔期/禁渔区警示 | 法律风险，需对接政府数据 |
| 5 | 举报 + 内容审核工作台 | 运营必须 |
| 6 | 手机号绑定 + 实名 | 发布前合规 |

### 🌟 强化 — 拉差异化

| # | 需求 | 备注 |
|---|---|---|
| 7 | AI 识鱼 | 上传时识别鱼种 + 估重 |
| 8 | 宜钓指数 v2 | 加入用户历史回归(同点过往鱼获 vs 当时天气) |
| 9 | 钓点热力图 | 地图层叠加近 30 天鱼获密度 |
| 10 | 天气趋势图 | 详情页 48h 气压/温度曲线 |
| 11 | 钓鱼日报/月报 | 自动生成战绩长图，朋友圈分享 |

### 🛠 工程化

| # | 需求 | 备注 |
|---|---|---|
| 12 | 草稿与断点续传 | 发布鱼获多图易丢 |
| 13 | 多端分享落地 H5 | 微信外打开 |
| 14 | 小程序分包 | 地图 + 上传相关分包加载 |
| 15 | 监控告警 + 灰度 | 上线后必备 |
| 16 | 数据看板 | DAU/发布率/留存 |

### 商业化(暂缓)

黑坑预订、比赛打卡、商城、付费教学等暂不纳入 v2，待 DAU/发布率达到运营临界值后再启动单独迭代。
